# Skill: Humanise prose

This skill rewrites prose so that it reads like natural technical writing.

It removes common AI writing patterns and increases information density.

# When to use

Use this skill when:

- editing AI-generated blog posts
- rewriting documentation
- cleaning up verbose prose
- improving readability

# Transformation goals

The rewrite should:

- remove rhetorical gimmicks
- simplify vocabulary
- reduce repetition
- eliminate filler transitions
- replace dramatic phrasing with precise statements

# Rewrite rules

Apply the following transformations.

## Simplify vocabulary

Replace inflated words.

Examples:

utilize -> use  
leverage -> use  
delve -> explain  
robust -> reliable

## Remove rhetorical structures

Rewrite patterns like:

The result? Devastating.

into:

This causes severe performance degradation.

## Remove suspense transitions

Delete phrases such as:

Here's the kicker  
Here's the thing  
Here's what most people miss

## Remove teaching tone

Rewrite:

Let's break this down.

into:

The mechanism works as follows.

## Reduce repetition

If the same idea appears in multiple paragraphs, keep the clearest explanation and remove the rest.

## Remove exaggerated claims

Rewrite statements such as:

This will change everything.

into:

This changes how the system handles dependency resolution.

# Desired output characteristics

The final text should be:

concise  
clear  
mechanistic  
informative

The reader should feel that a domain expert wrote it, not a marketing department.
